# 多信源数据校验协议

本文件定义 `Stock Analyzer` 的当前生效校验规则，用于在 API 取数之后，对关键字段做交叉验证、冲突仲裁和置信度披露。

本文件只讨论：

- 怎样校验数据
- 哪些字段必须重点校验
- 遇到冲突时怎么处理

本文只定义当前生效的数据校验规则，不讨论历史演变。

## 1. 与其他规则文件的关系

校验顺序：

1. 先按 `api-data-source-protocol.md` 完成 API 优先取数
2. 再按本文件做关键字段校验
3. 再按市场规则与行业规则解释这些数据

相关文件：

- `references/api-data-source-protocol.md`
- `references/a-share-market-rules.md`
- `references/hk-market-rules.md`
- `references/us-market-rules.md`
- `references/cyclical-industry-rules.md`
- `references/non-cyclical-industry-rules.md`

## 2. 总体原则

- 先核验，再分析
- 关键字段必须能回溯来源
- 数据类型必须明确，不能把不同含义的数字混用
- 跨市场标的必须独立取数，禁止直接换算推导
- 如果校验做不实，宁可降级披露，也不要假装确定

## 3. 先做两层分类

在校验前先判断：

1. 市场类型：A 股 / 港股 / 美股或 ADR
2. 行业类型：周期性 / 非周期性

作用：

- 市场类型决定币种、财报口径、分红习惯和治理重点
- 行业类型决定估值口径，尤其是能不能直接使用当期 PE

## 4. 头号红线：数据类型不能混淆

价格类数字必须严格区分以下身份：

| 数据身份 | 能否当“当前股价”使用 |
|---|---|
| 实时价 / 最新收盘价 | 可以 |
| 目标价 | 不可以 |
| 一致预期均值 | 不可以 |
| 52 周高低点 / 区间 | 不可以 |
| DCF / 内在价值估算 | 不可以 |
| IPO 价 / 历史价 | 不可以 |

任何报告如果把“目标价、历史区间、模型估值”写成现价，本次校验直接判失败。

## 5. 必校验字段

以下字段必须重点校验：

| 类别 | 字段 |
|---|---|
| 价格 | 当前股价、收盘价、总市值 |
| 估值 | PE、PB、PS、EV/EBITDA、历史分位 |
| 财务 | 营收、归母净利、毛利率、净利率、ROE |
| 回报 | 股息率、派息率、分红金额 |
| 杠杆 | 资产负债率、有息负债率 |
| 现金流 | 经营现金流、自由现金流 |
| 结构 | 股东结构、创始人或实控人持股 |
| 行业 | 市占率、行业地位、竞争格局 |

## 6. 校验强度

### 6.1 价格类字段

最低要求：

- 至少 1 个高置信 API 源
- 再补 1 到 2 个网页或官方源做交叉

重点防错：

- 币种混淆
- 实时价与收盘价混淆
- AH / ADR 错位
- 把目标价当现价

### 6.2 财务与估值字段

最低要求：

- 至少 1 个 API 或公告源
- 再补 1 个权威网页或数据库源

重点防错：

- TTM 与静态值混淆
- GAAP 与调整后利润混淆
- 加权 ROE 与摊薄 ROE 混淆
- 周期底部利润导致 PE 失真

### 6.3 定性字段

如行业格局、市占率、管理层言论、近 30 天事件：

- 至少 2 个独立来源
- 优先行业协会、监管公告、公司原文、权威媒体

## 7. 市场专项校验点

### 7.1 A 股

重点校验：

- 实控人、大股东、质押、减持、再融资
- 主营构成与分部口径
- 分红预案与实施公告是否一致
- 财年与累计单季口径是否混淆

### 7.2 港股

重点校验：

- HKD 币种
- AH 双重上市两端是否独立取数
- 高股息是否可持续
- 流动性折价与经营问题是否混淆

### 7.3 美股 / ADR

重点校验：

- GAAP 与调整后利润
- FCF 与回购
- 指引与实际业绩差异
- ADR 价格表现与底层经营是否错位

## 8. 行业专项校验点

### 8.1 周期性行业

必须补校验：

- 库存
- 开工率
- 产品价格
- 原材料价格
- 下游需求

估值时不得只看当期 PE，要优先验证：

- 正常化利润
- PB
- 周期中枢估值

### 8.2 非周期性行业

必须补校验：

- 用户或客户黏性
- 定价权持续性
- 护城河是否强化
- 高估值是否由高质量支撑

## 9. 字段级常见陷阱

### 9.1 股价与市值

常见问题：

- 用盘中价替代收盘价
- 用错误币种计算市值
- 用 ADR 或 A/H 另一地价格替代本地价格

### 9.2 PE 与历史分位

常见问题：

- 静态 PE 冒充 TTM
- 不说明分位周期
- 周期股底部直接用高 PE 判贵

### 9.3 营收与净利

常见问题：

- 财年错位
- 合并口径与母公司口径混淆
- 调整后利润与归母净利混淆

### 9.4 分红与股息率

常见问题：

- 中期和年末分红漏加
- 特别股息当成常态
- 美股回购与现金分红混成一个概念

### 9.5 现金流与负债

常见问题：

- FCF 定义不统一
- 把无息负债当有息负债
- 不披露租赁负债等表内变化

## 10. 冲突仲裁

当不同来源冲突时，按以下顺序仲裁：

1. 交易所或公司公告原文
2. 官方 API 或高置信结构化 API
3. 权威数据库网页
4. 财经媒体
5. 弱信源

仲裁规则：

- API 与网页冲突：优先 API，除非 API 明显异常
- API 与 API 冲突：优先交易所口径或公司公告
- 弱信源与强信源冲突：直接舍弃弱信源

## 11. 置信度披露

校验完成后，每个关键字段至少归入以下三类之一：

- 高置信度：多个高质量来源一致
- 中置信度：基本一致，但仍有轻微口径差异
- 低置信度：来源不足或存在明显冲突

低置信度字段：

- 不应直接支撑强结论
- 必须在报告中单独披露

## 13. 立即停止分析的情况

以下情况出现时，应暂停分析并回报：

- 关键价格字段身份不明
- 跨市场标的未独立取数
- 关键财务字段存在大幅冲突且无法仲裁
- 主要业务板块数据严重不全
- 结论依赖低置信度字段

## 14. 禁止行为

- 把网页目标价、区间值、估值值写成现价
- 不披露字段来源和冲突处理
- 用单日资金流向替代基本面校验
- 用单一来源支撑强投资结论
